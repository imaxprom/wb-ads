# WB Ads

## Проект
- Рабочая директория: `/Users/octopus/Projects/wb-ads`
- Стек: Next.js + TypeScript + Tailwind CSS + SQLite (better-sqlite3) + Puppeteer
- Порт: 3001
- БД: `data/ads.db` (25 таблиц)
- Контекст проекта: `PROJECT_CONTEXT.md`
- База знаний в UI: `KnowledgeBase.tsx` (13 разделов)

## Команды
- `npm run dev` — запуск dev-сервера на порту 3001
- `npm run init-db` — инициализация БД (создание всех таблиц)

## Правила
- ЗАПРЕЩЕНО обращаться к `/Users/octopus/Projects/website/` или другим проектам без явного разрешения пользователя
- Все зависимости локальные (`npm install` без `-g`)
- Проект полностью автономный, не зависит от MpHub
- Все даты через `localDateStr()` (Москва), НЕ `toISOString()` (UTC)
- Все настройки в БД через API (НЕ localStorage)
- SVG-иконки вместо эмодзи

## Сохранение контекста (ОБЯЗАТЕЛЬНО)
- При завершении сессии или по запросу пользователя — ОБЯЗАТЕЛЬНО сохранить:
  1. `PROJECT_CONTEXT.md` — обновить если были изменения в архитектуре, таблицах, формулах, sync
  2. Память (`~/.claude/projects/-Users-octopus-Projects-wb-ads/memory/`) — обновить project_wb_ads.md, user_preferences.md, wb_analytics_research.md
  3. `KnowledgeBase.tsx` — обновить если менялись столбцы, таблицы, формулы, настройки
- Если сессия длится больше 1 часа — промежуточное сохранение контекста
- Если пользователь говорит "запиши", "сохрани", "запомни" — немедленно сохранить в память
- При сохранении — НЕ использовать кэш, перечитывать код заново и сверять с реальным состоянием
